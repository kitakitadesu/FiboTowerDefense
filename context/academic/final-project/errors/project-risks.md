# Errors: Project Risks

**Core Idea**: Common pitfalls and critical requirements that impact grading.

**Gotchas**:
- **Q&A Risk**: Must link source code lines to specific rules/requirements during Q&A.
- **Class Count**: Minimum 2 *additional* classes required (don't count provided base classes).
- **Inheritance**: Must have at least 2 inheritance points from base classes.
- **State Mod**: Objects must be added/removed from collections at runtime.

**Reference**: Final Project Criteria PDF, Page 2
